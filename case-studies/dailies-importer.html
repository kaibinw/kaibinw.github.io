Dailies Importer

A DaVinci Resolve script that turns hours of daily dailies-timeline setup into one click: feed it the cut's EDL once, then every morning it scans the shot tree, picks up overnight renders by department, and appends them to the right review timeline with colour coding. Shown here in its core form; studio-specific integrations stay in-house.

ROLE Design, build, daily operation STACK Python · Resolve scripting API · JSON configs STATUS In daily studio use

The studio problem

Ask any VFX editor about their mornings. Overnight, artists publish renders into ShotGrid across hundreds of shots — animation blocking, lighting takes, comp versions, matte-painting passes — and before the day's first review, editorial has to find what's new, pull it in, and rebuild dailies timelines the supervisor can actually scrub. On a busy show that's hours of setup, every single day, multiplied by however many departments are reviewing. It's the most repetitive job in editorial and the most punishing to get wrong: a missed render is a shot that doesn't get reviewed, and everyone in the room notices.

The fix isn't importing faster. It's never rebuilding: conform the cut once, then only ever append what changed.

What it does

Takes an EDL onceThe cut comes in as an EDL a single time and becomes the master review timeline. That's the whole setup cost. From then on the tool maintains the timelines; nobody rebuilds them by hand again.
Scan by departmentWalks the show's render folders and classifies new media by which department published it: animation, lighting, compositing, matte painting, editorial temps. Frame sequences and movie files both.
Config as memoryDepartment render types live in a JSON config with an on/off toggle each. Newly discovered types are added disabled: the tool never imports something it hasn't been told to want.
Append, don't rebuildA timeline registry remembers which review timeline each department feeds. Every run appends only what's new — the hours of daily setup collapse into the seconds the import takes.
Colour as languageEvery imported clip gets a Resolve colour tag by department, so the timeline reads at a glance in review: who published it, and how far along it is.

What a run looks like

What it demonstrates

  • Editorial knowledge is the spec. The department breakdown, the timeline structure, and the colour language encode how reviews actually run. A pipeline TD could write this script; only an editor would know what it should do.
  • Config-driven means show-portable. Point it at a different show root, hand it that show's EDL, let it discover the render types, toggle on what matters. Nothing is hardcoded to one production.
  • Trust through restraint. Unknown media is surfaced but never auto-imported, and reruns are idempotent. The tool earns its place in a daily-deadline workflow by being boring and predictable.
