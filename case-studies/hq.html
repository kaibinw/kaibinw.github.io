HQ

A local control panel over my personal knowledge vault and its agentic tooling. Daily report, live todo panel, session history, and one-click buttons that run headless Claude Code automations with output streaming into the page.

ROLE Product, design, engineering direction STACK Single Node file · zero packages STATUS In daily use

The studio problem

Every pipeline accumulates powerful automation that only TDs can drive. A coordinator who needs a delivery re-verified, a QC pass re-run, or a status report generated has to file a ticket and wait for someone with a shell. The automation exists; the front door doesn't. What production actually needs is a wall of safe, curated buttons: one click, live output, an audit trail, and no way to break anything.

My household has the same shape: powerful agentic automation over a knowledge vault, and users who will never open a terminal.

The same problem at household scale

My second brain is a markdown vault operated through agentic sessions: ingest notes, audit the wiki, sort daily files. That works brilliantly if you live in a terminal, and not at all if you don't. The people who share the household don't open shells. The automations needed a front door: something a non-terminal user can open in a browser and press one button.

Constraints set before building

Zero dependenciesOne Node file, no node_modules, nothing to install or update. Any Node ≥18 runs it.
The vault is the databaseNo copy of the data anywhere. The panel renders the markdown files directly and writes checkbox ticks straight back to them.
Local onlyBinds to the local machine, never the network. Personal notes stay personal.
Nothing destructive one click awayButtons only run additive automations; the vault's hard rules (never delete, always log) hold underneath.

One click, visible consequence

The Run tab is the point of the tool. Each button is a stored prompt sent to a headless agent working inside the vault; the output streams live into the panel, and the run lands in the session log like any other. Curating the buttons is editing a small JSON file.

What it demonstrates

  • Agentic engineering as infrastructure, not novelty. The agent is a subprocess that any UI can wrap. HQ turns conversational automations into buttons a family member can press.
  • Constraints produce simplicity. "Zero dependencies" forced a single-file server; two years of npm churn will never touch it.
  • Editorial discipline transfers. The panel reads like a logbook because vault operations, like cut changes, deserve an audit trail.
